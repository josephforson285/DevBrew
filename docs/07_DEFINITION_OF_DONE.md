# 07 - Definition of Done (DVBRW-20)

A story is **Done** only when it satisfies its own acceptance criteria **and**
all of the following:

- The implementation satisfies all acceptance criteria for the story.
- The code is in the correct project structure.
- The code follows readable naming and formatting conventions (ruff clean).
- Relevant unit / integration tests are added where appropriate.
- All tests pass locally.
- The GitHub Actions CI pipeline passes successfully.
- The work is committed with a meaningful message referencing the Jira issue.
- No known critical defects remain.
- README / project documentation is updated when needed.
- The completed work can be demonstrated during Sprint Review.

No story moves to Done until it meets its acceptance criteria and this shared
Definition of Done.
